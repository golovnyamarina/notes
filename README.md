# Руководство программиста сделанно со Скрипицыной
***Бэкэнд***
1. Серверная часть 
Сервер отвечает за хранение данных, логику авторизации и предоставление API
Модель данных (Note): Содержит заголовок, текст, временные метки и связь с пользователем через ForeignKey. Каждая заметка привязана к своему автору
<img width="574" height="365" alt="image" src="https://github.com/user-attachments/assets/742822d5-744d-413f-9985-9422331a1f16" />

Авторизация: Используется библиотека SimpleJWT.
При логине сервер выдает два токена: access (краткосрочный, 60 мин) и refresh (долгосрочный, 1 день)
Все запросы к заметкам требуют access-токен в заголовке Authorization: Bearer <token>
Безопасность (IsOwner): Специальный класс разрешений проверяет, что редактировать или удалять заметку может только тот пользователь, который её создал.

***Фронтенд***
2. Клиентская часть 
Клиент обеспечивает интерфейс и взаимодействует с API через библиотеку Axios
Управление состоянием (useAuth): composable инкапсулирует логику входа, регистрации и хранения данных о пользователе. Он отслеживает, авторизован ли клиент в данный момент
Request Interceptor: Автоматически добавляет актуальный токен из localStorage в каждый исходящий запрос.
Response Interceptor: Если сервер вернул ошибку 401 (токен истек), интерцептор сам отправляет refresh-токен, получает новый access, сохраняет его и повторяет исходный запрос. Пользователь этого даже не замечает
Маршрутизация (Vue Router): Защищает страницы. Если пользователь не вошел в систему, его автоматически перекидывает на страницу /login при попытке открыть заметки 

***Взаимодействие***
3. Схема взаимодействия
Регистрация/Вход: Пользователь отправляет данные через форму. Сервер проверяет их и возвращает токены.
Хранение: Токены сохраняются в браузере (localStorage).
<img width="440" height="430" alt="image" src="https://github.com/user-attachments/assets/e2568259-9b53-422d-95c5-afd1f175c0a1" />

Запрос данных: При открытии списка заметок Vue делает GET-запрос. Сервер через get_queryset фильтрует базу и отдает только те заметки, которые принадлежат текущему пользователю.
<img width="493" height="331" alt="image" src="https://github.com/user-attachments/assets/433a7ec7-2275-4dd1-86fd-10a3037e13ae" />

CRUD (Создание, Чтение, Обновление, Удаление):
Пользователь нажимает «Сохранить» - фронтенд отправляет POST или PUT запрос.
Сервер валидирует данные через сериализаторы и обновляет БД.
